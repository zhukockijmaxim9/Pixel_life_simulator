# Ассеты и контент

Этот документ описывает, где лежат ресурсы (иконки/видео/шрифты) и как правильно добавлять новые.

## Где лежат ассеты

- `assets/`
  - `assets/icons/` — иконки для UI
  - `assets/merch/` — изображения мерча
  - `assets/characters/` — персонажи (изображения/видео)
  - `assets/fonts/` — кастомные шрифты

Реальный список ресурсов, подключённых к приложению, находится в `pubspec.yaml` в секции `flutter/assets`.

## Точка правды: `pubspec.yaml`

Flutter подхватывает ассеты только те, что перечислены в `pubspec.yaml`. Если файл лежит в папке, но не указан в `assets:`, он не будет доступен через `Image.asset(...)`.

## Как добавить новый ассет

1. Положите файл в подходящую папку внутри `assets/`.
2. Добавьте путь в `pubspec.yaml` (секция `flutter -> assets`).
3. Выполните:

```bash
flutter pub get
```

4. Используйте ассет в коде (например, `Image.asset(...)`).

## Картинки с разными разрешениями

Если вы добавляете картинки, можно использовать “варианты” (2.0x/3.0x) по стандарту Flutter:

- `assets/icons/my_icon.png`
- `assets/icons/2.0x/my_icon.png`
- `assets/icons/3.0x/my_icon.png`

Тогда Flutter автоматически выберет подходящее разрешение под устройство.

## Соглашения об именовании

- **только латиница**, нижний регистр, слова через `_`
- избегайте пробелов и кириллицы в именах файлов
- видео/изображения группируйте по назначению (`characters`, `icons`, `merch`)

## Видео (characters)

Если используете `video_player`, следите за:

- размером файлов (влияет на размер сборки),
- поддерживаемыми кодеками на целевых платформах,
- корректными путями в `pubspec.yaml`.

## Контент игры (данные)

Игровые сущности задаются данными в `lib/data/`:

- `jobs_data.dart` — профессии и условия доступа
- `goals_data.dart` — цели месяца
- `events_data.dart` — события и квизы
- `courses_data.dart` — курсы
- `merch_data.dart` — предметы магазина
- `game_data.dart` — сводный доступ (точка входа к данным)

Если вы добавляете новый контент, старайтесь:

- поддерживать уникальные `id` (для событий),
- сохранять баланс (цены/награды/штрафы),
- обновлять документацию правил/контента при изменениях.

## Чеклист при добавлении контента

- [ ] Добавили ассет в нужную папку `assets/...`
- [ ] Прописали путь в `pubspec.yaml`
- [ ] Проверили, что UI не ломается (запуск на целевой платформе)
- [ ] Если контент меняет механику — обновили `docs/gameplay.md`
